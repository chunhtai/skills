# write_documentation

Generate or improve documentation for code, APIs, or project components.

## Parameters

- **file_path** (string, required): Path to the file or directory to document.
- **doc_type** (string, optional): Type of documentation to produce (e.g., inline, readme, api, changelog).

## Instructions

Create clear, accurate documentation for the specified code:
1. Write concise descriptions for all public functions, classes, and modules
2. Document all parameters with their types, descriptions, and whether they are required
3. Document return values and thrown exceptions/errors
4. Include usage examples where helpful
5. Follow the documentation style already used in the project (e.g., JSDoc, docstrings, Godoc)
6. Keep documentation up to date with the current implementation

Good documentation explains *why* as well as *what*.
